# GlaMaterials

Інструмент, який накладає фірмовий водяний знак GlamEng на PDF або фото. Працює як PWA — можна додати на головний екран iPhone як окрему аппку.

Домен/бренд: **GlaMaterials** (GlamEng).

## Як це працює

1. Ти обираєш **шаблон** зверху: **«Шаблон»** (водяний знак поверх PDF-документів, значення `document`) або **«Рамка»** (вставляє фото/скріншот у фірмову рамку, значення `frame`). Перемикання шаблону **не** видаляє вже додані файли — вони автоматично переробляються під нове кадрування нового шаблону.
2. Ти перетягуєш або обираєш файли — PDF (будь-яка кількість сторінок), фото (JPG/PNG/WEBP) або HEIC/HEIF (формат iPhone). Файли можна додавати накопичувально, по одному — черга не скидається, поки ти сама не натиснеш «Очистити все».
3. **«Шаблон»**: кожна сторінка/фото вписується в аркуш A4 повністю, без обрізання (масштаб "contain"; якщо пропорції не A4 — з боків/зверху-знизу додаються білі поля, щоб шаблон водяного знаку ліг рівно). Поверх накладається водяний знак — прозорий "вирізаний" PNG (`assets/watermark-cutout.png` → `assets/watermark-data.js`), де видно тільки контрастні елементи (напис GlamEng по кутках і емблему), а не суцільну рожеву заливку. Прозорість регулюється повзунком (за замовчуванням 35%), прев'ю оновлюється миттєво.
4. **«Рамка»**: фото/скріншот вписується у вікно фірмової рамки (`assets/frame-cutout.png` → `assets/frame-data.js`) на всю ширину/висоту, обрізаючи зайве по краях (масштаб "cover" — як `object-fit: cover`), а не стискаючи із білими полями. Рамка накладається на повній непрозорості — без повзунка.
5. Якщо додано кілька файлів (і формат результату — PDF) — можна вибрати: **об'єднати все в один PDF** чи **зробити окремий файл на кожен завантажений документ** (тоді результат — ZIP з кількома PDF).
6. Формат результату: **PDF** (стандартний) або **Фото** — кожна сторінка окремим JPEG. На телефоні/планшеті (реальний тачскрін) відкриється системне вікно "Поділитися" з прямим збереженням у Фото; на десктопі (миша/трекпад, без тачскріна) — завжди прямий download або ZIP-архів з фото, без системного вікна "Поділитися". Для шаблону «Рамка» формат за замовчуванням — «Фото» (типовий випадок — контент для соцмереж).
7. Останні готові файли зберігаються в **Історії** (іконка годинника вгорі) — можна перезавантажити ще раз, без повторної обробки. Зберігається до 40 останніх результатів, старіші автоматично видаляються.
8. Прев'ю не обмежене першою сторінкою — стрілками ‹ › можна погортати всі сторінки з усіх доданих файлів (наскрізна нумерація, бейдж показує «2/5» і т.д.), щоб перевірити весь пакет перед завантаженням.
9. Прозорість водяного знаку запам'ятовується (`localStorage`) — наступного разу застосунок відкриється з тим значенням, яке ти лишила минулого разу, а не завжди з дефолтних 35%.
10. По завершенню завантаження — маленьке конфеті біля кнопки (`celebrate()` в `app.js`), просто для настрою.

**Все відбувається локально в браузері.** Файли ніколи нікуди не надсилаються — немає бекенду, немає зовнішнього сховища. Історія лежить у IndexedDB твого браузера на цьому пристрої.

## Бренд-система

GlaMaterials — частина сімʼї GlamEng (сайт [glameng.online](https://glameng.online)), тож дизайн узятий безпосередньо з токенів сайту, а не вигаданий окремо:

- **Кольори:** `--ink #3D1F2D`, `--champagne #E8849A`, `--blush #F2A0B8`, `--cream #FFF5F8` — ті самі змінні, що й у `glameng/index.html`. Магента лого-марки `#9D2F7F` — окремий "іконковий" колір, він і на `favicon.png`/`apple-touch-icon.png` сайту.
- **Шрифти:** display — Cormorant Garamond (лого, «Історія»), body — Inter. Самохостяться в `assets/fonts/` (тільки latin+cyrillic підмножини — досить для української), підключені через `assets/fonts/fonts.css`. Ніяких запитів до Google Fonts у рантаймі.
- **Вордмарк:** "Gla" (ink) + курсивне "Materials" (champagne) — той самий патерн, що "Glam" + курсивне "Eng" на сайті.
- **Лого-марка:** кругла емблема з розкритою книжкою — растрове зображення `assets/logo-mark.png` (1024×1024, прозорий фон), джерело — оригінал наданий власником бренду. Всі іконки (favicon/apple-touch-icon/192/512/maskable) генеруються з нього.
- **Візуальна глибина:** два розмиті кольорові "блоби" на фоні (`body::before/::after`) повільно дрейфують (`@keyframes blobFloatA/B`, ~25с цикл), картки/панелі з'являються через `fadeSlideUp`, кнопка «Завантажити» має градієнт + м'яке дихання тіні + смужку відблиску при наведенні, іконка лого мерехтить (`sparkleTwinkle`). Все зупиняється через `@media (prefers-reduced-motion: reduce)`.

## Структура

- `index.html`, `styles.css`, `app.js` — весь застосунок.
- `assets/watermark.jpg` — оригінальний шаблон водяного знаку A4 (джерело, у рантаймі не використовується).
- `assets/watermark-cutout.png` — той самий шаблон, оброблений через прозорість (фон вирізаний, лишились тільки контрастні елементи) — джерело для `watermark-data.js`.
- `assets/watermark-data.js` — `watermark-cutout.png`, вшитий як base64 (щоб не було зовнішніх запитів).
- `assets/frame.png` — оригінальний шаблон рамки для скріншотів, landscape A4 (джерело, у рантаймі не використовується).
- `assets/frame-cutout.png` — той самий шаблон з прозорим вікном по центру (там, де показується фото) — джерело для `frame-data.js`.
- `assets/frame-data.js` — `frame-cutout.png`, вшитий як base64.
- `assets/logo-source.html` — вихідний SVG лого-марки (кругла емблема з книжкою), прозорий фон. `assets/logo-source-maskable.html` — той самий значок, але з меншим відступом і білим тлом, під Android-safe-zone (маскабельні іконки).
- `assets/icons/` — усі растеризовані іконки для PWA (favicon, apple-touch-icon, 192/512, maskable), згенеровані з `logo-source*.html`.
- `assets/fonts/` — self-hosted Cormorant Garamond + Inter (woff2, latin+cyrillic) і `fonts.css` з `@font-face`.
- `manifest.json`, `sw.js` — PWA-маніфест і service worker (кешування статики, "Додати на екран Головна" на iPhone).
- `vendor/` — `pdf-lib` (збірка PDF), `pdf.js` (рендер вхідних PDF-сторінок), `heic2any` (конвертація HEIC → JPEG), `jszip` (архіви) — усе завантажено локально, без CDN у рантаймі. `heic2any` і `jszip` підвантажуються лише за потреби (лениво), щоб не роздувати початкове завантаження сторінки.
- `vercel.json` — кеш-заголовки для статики.

Ніяких залежностей для збірки — це чистий статичний сайт, працює просто відкриттям `index.html` або з будь-якого статичного хостингу.

## Локальний запуск

```bash
python3 -m http.server 8080
# відкрити http://localhost:8080
```

## Деплой на Vercel

Найпростіше — так само, як інші сайти в цьому акаунті: підключити GitHub-репозиторій до Vercel, і кожен push у `main` буде деплоїтись автоматично.

1. Створити репозиторій на GitHub (наприклад `glamaterials`) і запушити цей код.
2. На [vercel.com](https://vercel.com) → **Add New Project** → обрати цей репозиторій.
3. Framework Preset: **Other** (це статичний сайт, білд не потрібен — Build Command і Output Directory можна лишити порожніми).
4. Deploy.

Після цього Vercel видасть посилання на кшталт `glamaterials.vercel.app`, яким можна одразу ділитися. За бажанням — прив'язати кастомний домен через Vercel → Settings → Domains.

**Важливо для iPhone:** Web Share API (пряме збереження фото в Галерею) і "Додати на екран Домівки" коректно працюють лише на **https** (або localhost). На проді (Vercel) це вже так з коробки.

## Додати на головний екран iPhone

Safari → відкрити сайт → кнопка "Поділитися" → **"На екран «Домівка»"**. З'явиться іконка GlaMaterials, застосунок відкриється на весь екран, без адресного рядка.

## Налаштування за замовчуванням

- Прозорість водяного знаку: змінна `DEFAULT_OPACITY` у `app.js` (зараз `35`, тобто 35%).
- Ліміт історії: `MAX_HISTORY` у `app.js` (зараз `40` файлів; кожен результат, більший за `MAX_HISTORY_ITEM_BYTES` — 30 МБ — в історію не зберігається, хоча сам файл все одно завантажується).
- Ліміт файлів за раз на вхід: `MAX_JOBS` у `app.js` (зараз `40` файлів у черзі одночасно; кожен PDF-файл усередині може мати будь-яку кількість сторінок — окремого ліміту на сторінки немає).

## Перегенерувати водяний знак

Якщо потрібно змінити сам шаблон — постав новий `assets/watermark.jpg` (A4, портрет) і перезапусти вирізання прозорості (Python/Pillow, дивись історію розробки): фон близький до домінантного кольору стає прозорим, контрастні елементи (текст/лінії) лишаються й перефарбовуються в акцентний рожевий. Результат зберегти як `assets/watermark-cutout.png`, потім перегенерувати `assets/watermark-data.js` (base64 PNG data URI).

## Перегенерувати рамку для скріншотів

Постав новий `assets/frame.png` (landscape A4, 2000×1414) і виріж прозоре вікно там, де має показуватись фото — прямокутник координат зараз `{x: 291, y: 216, w: 1397, h: 961}` (заданий у `TEMPLATES.frame.window` в `app.js`, підібраний під поточний дизайн рамки). Якщо новий дизайн має вікно в іншому місці/розмірі — поміняй ці координати в `app.js` відповідно до нового `frame.png`. Результат зберегти як `assets/frame-cutout.png`, потім перегенерувати `assets/frame-data.js`.

## Перегенерувати лого / іконки

1. Постав новий вихідний файл лого (квадратне зображення, контрастний фон) — заміни ним `assets/logo-mark.png` вручну, або оброби через Pillow: різниця кольору від фону → альфа-канал, обрізка по bbox з відступом, композит у квадратний канвас, ресайз до 1024×1024 (`LANCZOS`).
2. З отриманого `assets/logo-mark.png` вирізати всі розміри іконок (192/512/apple-touch/favicon/maskable) — та сама Pillow-логіка: композит на білий/прозорий канвас з потрібним відступом під кожен розмір.
3. `index.html` підключає лого напряму як `<img class="brand-mark" src="assets/logo-mark.png">` у шапці — нічого редагувати там не треба, він підхопить новий файл автоматично.
4. Не забудь підняти версію `CACHE` у `sw.js` — іконки кешуються cache-first, інакше користувачі, які вже відкривали застосунок, побачать старе лого ще довго.
5. **Обов'язково постав/підніми `?v=N` у query-параметрі на всіх посиланнях на іконки** — `assets/icons/apple-touch-icon.png?v=2`, `favicon.png?v=2`, `logo-mark.png?v=2` в `index.html`, і всі три `src` в `manifest.json`. Причина: `vercel.json` віддає все з `/assets/` як `Cache-Control: public, max-age=31536000, immutable` — браузер (особливо iOS Safari при "Додати на Домівку") закешовує файл за URL НАЗАВЖДИ і навіть не перевіряє, чи він змінився на сервері. Без зміни URL (нового `?v=`) заміна файлу під тим самим іменем ніколи не долетить до вже відкритих телефонів — саме тому іконка на iPhone лишалась старою після попередніх релогування лого. Просте підняття версії sw.js `CACHE` тут НЕ допомагає — це кеш самого service worker'а, а `apple-touch-icon`/`favicon`/`manifest`-іконки iOS запитує напряму по URL з `<link>`/manifest, минаючи SW.
