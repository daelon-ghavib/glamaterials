# GlaMaterials

Інструмент, який накладає фірмовий водяний знак GlamEng на PDF або фото. Працює як PWA — можна додати на головний екран iPhone як окрему аппку.

Домен/бренд: **GlaMaterials** (GlamEng).

## Як це працює

1. Ти перетягуєш або обираєш файли — PDF (будь-яка кількість сторінок), фото (JPG/PNG/WEBP) або HEIC/HEIF (формат iPhone). Файли можна додавати накопичувально, по одному — черга не скидається, поки ти сама не натиснеш «Очистити все».
2. Кожна сторінка/фото вписується в аркуш A4 (масштаб "contain" — без спотворень; якщо пропорції не A4, з боків/зверху-знизу додаються білі поля, щоб шаблон водяного знаку ліг рівно на кожній сторінці).
3. Поверх накладається водяний знак — прозорий "вирізаний" PNG (`assets/watermark-cutout.png`, вшитий у `assets/watermark-data.js` як base64), де видно тільки контрастні елементи (напис GlamEng по кутках і емблему), а не суцільну рожеву заливку. Прозорість регулюється повзунком (за замовчуванням 35%), прев'ю оновлюється миттєво.
4. Якщо додано кілька файлів — можна вибрати: **об'єднати все в один PDF** чи **зробити окремий файл на кожен завантажений документ** (тоді результат — ZIP з кількома PDF).
5. Формат результату: **PDF** (стандартний) або **Фото в галерею** — кожна сторінка окремим JPEG; на iPhone відкриється системне вікно "Поділитися" з прямим збереженням у Фото, на інших пристроях — ZIP-архів з фото.
6. Останні готові файли зберігаються в **Історії** (іконка годинника вгорі) — можна перезавантажити ще раз, без повторної обробки. Зберігається до 8 останніх результатів, старіші автоматично видаляються.

**Все відбувається локально в браузері.** Файли ніколи нікуди не надсилаються — немає бекенду, немає зовнішнього сховища. Історія лежить у IndexedDB твого браузера на цьому пристрої.

## Структура

- `index.html`, `styles.css`, `app.js` — весь застосунок.
- `assets/watermark.jpg` — оригінальний шаблон A4 (джерело, у рантаймі не використовується).
- `assets/watermark-cutout.png` — той самий шаблон, оброблений через прозорість (фон вирізаний, лишились тільки контрастні елементи) — джерело для `watermark-data.js`.
- `assets/watermark-data.js` — `watermark-cutout.png`, вшитий як base64 (щоб не було зовнішніх запитів).
- `assets/icons/` — іконки для PWA (favicon, apple-touch-icon, 192/512, maskable).
- `manifest.json`, `sw.js` — PWA-маніфест і service worker (кешування статики, "Додати на екран Головна" на iPhone).
- `vendor/` — `pdf-lib` (збірка PDF), `pdf.js` (рендер вхідних PDF-сторінок), `heic2any` (конвертація HEIC → JPEG), `jszip` (архіви) — усе завантажено локально, без CDN у рантаймі. `heic2any` і `jszip` підвантажуються лише за потреби (лениво), щоб не роздувати початкове завантаження сторінки.
- `vercel.json` — кеш-заголовки для статики.

Ніяких залежностей для збірки — це чистий статичний сайт, працює просто відкриттям `index.html` або з будь-якого статичного хостингу.

## Локальний запуск

```bash
python3 -m http.server 8080
# відкрити http://localhost:8080
```

## Деплой на Vercel

Найпростіше — так само, як інші сайти в цьому акаунті: підключити GitHub-репозиторій до Vercel, і кожен push у `main` буде деплоїтись автоматично.

1. Створити репозиторій на GitHub (наприклад `glamaterials`) і запушити цей код.
2. На [vercel.com](https://vercel.com) → **Add New Project** → обрати цей репозиторій.
3. Framework Preset: **Other** (це статичний сайт, білд не потрібен — Build Command і Output Directory можна лишити порожніми).
4. Deploy.

Після цього Vercel видасть посилання на кшталт `glamaterials.vercel.app`, яким можна одразу ділитися. За бажанням — прив'язати кастомний домен через Vercel → Settings → Domains.

**Важливо для iPhone:** Web Share API (пряме збереження фото в Галерею) і "Додати на екран Домівки" коректно працюють лише на **https** (або localhost). На проді (Vercel) це вже так з коробки.

## Додати на головний екран iPhone

Safari → відкрити сайт → кнопка "Поділитися" → **"На екран «Домівка»"**. З'явиться іконка GlaMaterials, застосунок відкриється на весь екран, без адресного рядка.

## Налаштування за замовчуванням

- Прозорість водяного знаку: змінна `DEFAULT_OPACITY` у `app.js` (зараз `35`, тобто 35%).
- Ліміт історії: `MAX_HISTORY` у `app.js` (зараз `8` файлів).
- Ліміт файлів за раз: `MAX_JOBS` у `app.js` (зараз `40`).

## Перегенерувати водяний знак

Якщо потрібно змінити сам шаблон — постав новий `assets/watermark.jpg` (A4, портрет) і перезапусти вирізання прозорості (Python/Pillow, дивись історію розробки): фон близький до домінантного кольору стає прозорим, контрастні елементи (текст/лінії) лишаються й перефарбовуються в акцентний рожевий. Результат зберегти як `assets/watermark-cutout.png`, потім перегенерувати `assets/watermark-data.js` (base64 PNG data URI).
